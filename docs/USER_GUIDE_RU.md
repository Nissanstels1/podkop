# Podkop v2 — Руководство для чайников

Этот файл — пошаговое объяснение для тех, кто ставит podkop впервые. Если
вы уже знаете, что такое sing-box и как работает urltest — листайте сразу
к разделу «Тонкая настройка».

---

## 1. Что вообще такое подписка

«Подписка» — это ссылка вида `https://my-panel.example.com/sub/abcdef`,
по которой ваш VPN-провайдер отдаёт *список серверов*. Браузер по такой
ссылке покажет либо набор `vless://...` строк, либо файл в формате Clash
YAML, либо base64-крякозябру — всё это варианты одного и того же.

Podkop ходит за этой ссылкой по расписанию (по умолчанию — раз в сутки),
парсит список, превращает его в конфиг для sing-box и применяет.

---

## 2. Как добавить подписку через LuCI

1. Откройте веб-интерфейс роутера → **Network → Podkop**.
2. На вкладке **Sections** должна быть секция «main». Если её нет — нажмите «Add».
3. В поле **Connection Type** выберите `proxy`.
4. В поле **Configuration Type** выберите `subscription`.
5. Появится поле **Ссылки на подписки** (Subscription URLs). Нажмите `+`
   (плюс) и вставьте вашу ссылку. Если у вас несколько провайдеров —
   нажмите `+` ещё раз и добавьте вторую ссылку.
6. Нажмите **Save & Apply** внизу страницы.

После Save & Apply podkop:
- сохраняет вашу ссылку в `/etc/config/podkop`;
- автоматически проверяет, изменился ли список URL по сравнению с прошлым
  состоянием — если да, **сразу** идёт за подпиской и парсит её (не нужно
  ждать кронов и не нужно жать «Update subscription now»);
- собирает sing-box конфиг и запускает сервис.

Если что-то пошло не так — `logread -e podkop` покажет подробности.

## 3. Поля секции — что за что отвечает

### Subscription URLs (Ссылки на подписки)
Главное поле. По одной ссылке в строке. Поддерживается:
- список `vless://`/`ss://`/`trojan://`/`hy2://` (по строке на сервер);
- то же, но в base64 (как у большинства панелей);
- Clash / clash.meta YAML;
- v2ray JSON (с полем `outbounds`);
- полный sing-box JSON.

Формат определяется автоматически. Использовать `http://` (не `https://`)
можно, но в логе будет предупреждение — DNS запроса и тело ответа в этом
случае идут открытым текстом.

### Subscription Format (Формат подписки)
Обычно — `Автоопределение`. Меняйте только если автодетект ошибся (это
видно в логе по строке `format=...`).

### User-Agent
Некоторые панели по разному отдают подписку для разных клиентов. Если у вас
sing-box-only панель — подпишитесь на `SFA/1.11.9`. По умолчанию `Auto`,
этого почти всегда хватает.

### Selection Mode (Режим выбора)
- **URLTest** — sing-box сам пингует все профили и подключает к самому
  быстрому. Это то, что нужно почти всем.
- **Selector** — первый профиль из списка. Переключение только вручную
  через Clash dashboard.

### Tag Filters (include / exclude) — Фильтр по тегам
Каждый профиль подписки имеет «тег» — обычно это название страны и эмодзи
флага: `🇩🇪 Germany #1`. Сюда можно вписать подстроки (через `|` для «или»
внутри одной строки):
```
NL|DE|FI         # любой тег с подстрокой NL ИЛИ DE ИЛИ FI
🇩🇪|🇳🇱           # эмодзи тоже работают
```
Несколько строк — тоже «или». Если оставить пусто — берутся все профили.

### Auto-update Interval (Интервал автообновления)
Как часто podkop сам идёт за подпиской.

**Рекомендуется `Раз в сутки`** — подписки редко меняются чаще.

Что важно понимать про обновление:
- На каждом обновлении podkop сравнивает sha256 нового тела подписки
  с предыдущим. **Если содержимое не изменилось — sing-box НЕ
  перезагружается.** Никаких «дёрганий» интернета.
- Если содержимое реально изменилось (новые серверы, удалены старые) —
  один аккуратный перезапуск sing-box. Активные TCP-соединения переподключатся,
  это короткий blip ~1-2 секунды.

### Allow Insecure TLS
Принимать самоподписанные / просроченные TLS-сертификаты при загрузке
подписки. Включайте, только если доверяете источнику.

## 4. Фильтр по диапазону пинга

> Сценарий: «у меня в подписке 1500 серверов, среди них есть российские
> с пингом 50 мс. Я хочу автоматически выбрать ИНОСТРАННЫЙ — поэтому
> чтобы российские в выбор НЕ попадали».

Поля:

### Min Latency (Минимальный пинг, мс)
Профили с пингом меньше этого числа отбрасываются. Для нашего сценария
поставьте `200` — все российские (50–80 мс) выпадут.

### Max Latency (Максимальный пинг, мс)
Профили с пингом больше этого числа отбрасываются. Например `500` —
выкинет совсем медленные.

### Что делать с свежеподключённой подпиской
Сразу после добавления подписки у podkop ещё нет измерений пинга. В этот
момент **фильтр НЕ применяется** к таким «непомеренным» профилям —
чтобы в первый запуск интернет вообще работал. Через несколько минут
(когда отработает сборщик пинга) фильтр применится полностью.

Чтобы ускорить — нажмите **Refresh latency cache** (Обновить кэш пинга)
на странице секции.

---

## 5. Быстрый / медленный пул (failover)

> Сценарий: «у меня большая подписка, основные быстрые серверы иногда
> отваливаются — хочу чтобы при их падении интернет НЕ пропадал, а
> переезжал на медленные».

Включите `Split into fast / slow pools` (Разделять на быстрый/медленный
пулы). Поставьте порог `Fast pool threshold (ms)` — например `100`.

Что произойдёт:
- profiles с пингом < 100мс уходят в `<секция>-fast`,
- остальные — в `<секция>-slow`,
- по умолчанию используется быстрый пул,
- раз в минуту watchdog проверяет, что хоть один профиль в быстром пуле
  жив. Если ВСЕ умерли — переключает на медленный (через Clash API,
  **БЕЗ перезапуска sing-box**, без обрыва соединений).

Когда хоть один быстрый профиль ожил — переключение обратно тоже
автоматическое.

## 6. DNS через прокси (защита от DNS-leak)

По умолчанию ваш роутер резолвит домены через DNS-сервер провайдера. Это
значит провайдер видит, к каким сайтам вы обращаетесь — даже если сам
трафик идёт через VPN.

Для перечисленных в `DNS-over-proxy domains` доменов podkop будет
резолвить ИХ через прокси (через DNS, который доступен на конце
VPN-туннеля), а не напрямую.

Пример типового набора:
```
chatgpt.com
openai.com
youtube.com
duckduckgo.com
```
Поддомены подцепятся автоматически.

Если прокси-DNS не ответил за `DNS-over-proxy timeout` (по умолчанию
2000 мс) — запрос летит к локальному резолверу, чтобы интернет не
тормозил.

## 7. Кнопки на странице секции

- **Update subscription now** — заставить podkop *сейчас* пойти за
  подпиской и перезапустить sing-box. Полезно сразу после добавления
  новой подписки.
- **Show parsed profiles** — открыть таблицу всех профилей, что были
  распарсены. Видно: тег, тип, эндпоинт, измеренный пинг, в каком пуле
  (fast/slow), прошёл ли фильтр (галочка ✓).
- **Refresh latency cache** — заставить sing-box пересчитать пинги
  через Clash API и применить фильтр заново.

## 8. Где смотреть, что происходит

Все события podkop пишет в системный журнал OpenWrt.

```
ssh root@router
logread -e podkop -f         # follow в реальном времени
logread -e podkop | tail -50 # последние 50 строк
```

Полезные строки:
```
subscription[main]: fetching https://...                  # пошёл за подпиской
subscription[main]: parsed 150 profiles                   # успешно распарсил
subscription[main]: upstream content unchanged            # ничего не изменилось, без перезагрузки
subscription[main]: upstream content changed, will reload # есть изменения, перезагрузка
subscription[main]: URL set changed, fetching now         # пользователь добавил новую ссылку
subscription[main]: pool watchdog: fast pool dead, switching to slow  # failover
```

## 9. Типичные сценарии конфигурации

### A. «Просто работает» (новичок)
- Subscription URLs: одна ссылка от провайдера.
- Auto-update Interval: `Раз в сутки`.
- Selection Mode: `URLTest`.
- Всё остальное — по умолчанию.

→ Будет автоматически выбираться сервер с лучшим пингом, без вашего
участия.

### B. «Только зарубежные»
- Subscription URLs: одна ссылка.
- Min Latency: `200`.
- Max Latency: `500`.

→ Все близкие серверы (включая российские) отвалятся, sing-box выберет
лучший среди оставшихся.

### C. «Большая подписка с failover»
- Subscription URLs: одна или несколько ссылок.
- Split into fast / slow pools: `1` (включено).
- Fast pool threshold: `120`.
- Auto-update Interval: `Раз в сутки`.

→ Основной трафик через быстрые серверы. Если их выкосит — автоматический
переезд на медленные без обрыва интернета.

### D. «Несколько провайдеров»
- Subscription URLs: добавьте несколько ссылок (от разных провайдеров).
- Дубли (по тегу + сервер:порт) удалятся автоматически.
- При обновлении одной из подписок sing-box перезагрузится один раз
  с обновлённым общим пулом.

## 10. Если что-то не работает

1. **Сохранить форму не получается, ругается на «Subscription URLs»**
   - Проверьте, что вы НАЖАЛИ на `+` и URL действительно появился в
     списке (а не просто набрали в поле).
   - Проверьте, что URL начинается с `https://` (или `http://`).

2. **Сохранил, но `Не удалось обновить подписку (код выхода=1)`**
   - Откройте `logread -e podkop` — там будет точная причина.
   - Самые частые: панель не отвечает (timeout), сертификат просрочен
     (включите Allow Insecure TLS если доверяете), формат не определился
     (попробуйте указать вручную).

3. **Подписка загружается, но после применения нет интернета**
   - Откройте `Show parsed profiles` — есть ли вообще выбранные (с ✓)?
   - Если фильтр пинга слишком жёсткий — все могут быть отфильтрованы.
     Поставьте Min/Max Latency = 0 чтобы временно отключить фильтр.
   - `sing-box check -c /etc/sing-box/config.json` покажет, валидный ли
     конфиг.

4. **Очень частый «дёргание» интернета**
   - Это ошибка. С v2r3+ подписка должна перезагружать sing-box только
     при реальном изменении содержимого. Если видите частые перезагрузки
     при стабильной подписке — пришлите `logread -e podkop` в issue.
